### Download

The Windows versions of **xPack { customField('appName') }**
are packed as `.zip` files.
Download the latest version named like:

* <code>xpack-{ customField('appLcName') }-{ customField('upstreamVersion') }-{ customField('xpackSubversion') }-win32-x64.zip</code>

:::note

In case you wonder where the suffix comes
from, it is exactly the Node.js `process.platform` and `process.arch`.
The `win32` part is confusing, but we have to live with it.

:::

### Unpack

To manually install the <b>xPack { customField('appName') }</b>,
unpack the archive and move it to the
<code>%USERPROFILE%\\AppData\\Roaming\\xPacks\\{ customField('appLcName') }\\xpack-{ customField('appLcName') }-{ customField('upstreamVersion') }-{ customField('xpackSubversion') }</code> folder
(for example <code>C:\\Users\\ilg\\AppData\\Roaming\\xPacks\\{ customField('appLcName') }\\xpack-{ customField('appLcName') }-{ customField('upstreamVersion') }-{ customField('xpackSubversion') }</code>);
according to Microsoft, <code>AppData\\Roaming</code> is the recommended location for
installing user specific packages.
